# アイアルク ブログ/クラファン制作補助アプリ 実装仕様書

> この文書は、VS Code上のClaude Codeが本リポジトリで実装作業を行う際の仕様書です。
> GitHub Actionsでのデプロイを前提とした構成・ファイル配置を想定しています。

## 0. スコープ（重要）

会社アカウントでの作業のため、**今回の実装範囲はブログ記事生成とクラウドファンディング本文生成の制作補助に限定**します。以下は将来構想として設計には織り込みますが、**今回は実装しません**。

- YouTube解説動画の台本・スライド生成
- 漫画形式の出力
- 小説形式の出力
- 動画生成

実装対象は次の一連の流れのみです。

口述筆記の入力・記録 → 要点抽出 → ブログ記事 or クラファン本文への整形 → DALL-Eによるイメージ画像・スライド風画像の生成

## 1. システム構成

```
[フロントエンド: HTML + TypeScript（べた書き、フレームワーク不使用）]
   ├─ 入力画面（タイムライン風UI、詳細は5章）
   ├─ 下書き一覧画面
   ├─ 生成結果編集画面（ブログ/クラファン切替）
   └─ 画像生成画面（DALL-Eプロンプト編集UI）
        │ HTTPS (fetch)
        ▼
[バックエンド: Cloud Run / Python / FastAPI]
   ├─ /drafts           下書きCRUD
   ├─ /generate/blog     ブログ整形 (Claude API)
   ├─ /generate/crowdfunding  クラファン本文整形 (Claude API)
   └─ /generate/image    画像生成 (DALL-E API)
        │
        ▼
[Firestore]
   ├─ tenant_settings
   ├─ drafts
   ├─ generated_texts
   └─ generated_images
```

フロントエンドはFirebase Hostingで配信、バックエンドはCloud Runにデプロイする想定。GitHub Actionsで両方のデプロイを自動化する。

## 4. プロンプト設計

2段階構成：(1) 要点抽出 → (2) 出力フォーマットへの整形。`{変数}` は3.0節の `tenant_settings` から埋め込む。

### 4.1 要点抽出プロンプト（`backend/prompts/extract_points.py`）

```
あなたは{org_name}のブログ・広報担当のアシスタントです。
以下は口述筆記された原文です。話し言葉特有の言い淀みや脱線を整理し、
箇条書きで要点を抽出してください。

# 出力形式
- 主題（1行）
- 要点（箇条書き、5〜8個）
- 印象的なエピソードや具体的な発言があれば、そのまま引用候補としてリストアップ

# 原文
{raw_text}
```

### 4.2 ブログ整形プロンプト（`backend/prompts/blog.py`）

```
あなたは{org_name}（{org_description}）の
ブログ記事を書くライターです。以下の要点をもとに、保護者が読みやすい
ブログ記事を作成してください。

# トーン
- {blog_tone}

# 構成
- タイトル
- 導入（1〜2段落）
- 本文（小見出しを使って読みやすく）
- まとめ（{blog_cta}）

# 要点
{extracted_points}
```

### 4.3 クラファン本文整形プロンプト（`backend/prompts/crowdfunding.py`）

```
あなたは{crowdfunding_project_name}のクラウドファンディング担当です。
以下の要点をもとに、プロジェクトページに掲載する文章を作成してください。

# 注意事項
- {crowdfunding_background}
- {crowdfunding_notes}
- 支援者の共感を呼ぶストーリー性を重視

# 要点
{extracted_points}
```

### 4.4 画像生成プロンプトのたたき台（`backend/prompts/image_prompt.py`）

```
以下のブログ/クラファン本文の要点から、DALL-Eに渡す画像生成プロンプトの
たたき台を1つ作ってください。{image_type}（スライド風 or イメージ画像風）
に適した構図・雰囲気を英語で記述してください。

要点: {extracted_points}
画像タイプ: {image_type}
```

生成されたプロンプトは初期値としてフロントエンドのテキストエリアに表示し、本人が編集してから `/generate/image` に送信する。AIが自動で最終画像を確定させない。

## 5. フロントエンドUI仕様

### 5.1 入力画面（タイムライン風）

- 音声認識（Web Speech API）またはテキスト直接入力で、話した/打った内容を**上から下に流れるチャット風のタイムライン**として表示する。
- 各発話の確定ごとに、断片を画面下部から薄く浮かび上がらせ、少し経ってから不透明度を下げて「過去のもの」として残す（チップ表示）。
- **沈黙検知トリガー**: 発話が止まってから2〜3秒経過したら、それまでの未処理の断片をまとめてバックエンドの要点抽出APIに送信する。毎発話ごとにAPIを呼ばない。
- 抽出された要点は、画面右側（またはタイムライン下部）にリアルタイムで追記表示する。印象的な発言は別枠（強調色）で「引用候補」として表示する。
- 一定量たまったら「下書きとして保存」「ブログ生成へ」のボタンを表示する。

### 5.2 下書き一覧画面

- `drafts` を一覧表示。各下書きから「ブログ生成」「クラファン生成」ボタンで `/generate/blog` または `/generate/crowdfunding` を呼び出す。

### 5.3 生成結果編集画面

- 生成された本文をMarkdown編集可能な形で表示。タブまたはトグルでブログ/クラファンを切り替え。
- 「画像生成へ進む」ボタン。

### 5.4 画像生成画面

- 4.4節のプロンプトをテキストエリアに初期表示し、本人が自由に編集してから生成ボタンを押す。
- **画像の「ぼかし→クリア」演出のタイミング設計（重要）**: 生成リクエスト送信と同時にプレースホルダー画像のぼかしを徐々に薄くし始めるが、**実際のAPIレスポンスが届くまでは完全にクリアな状態にしない**（演出が実体験を追い越さないようにする）。レスポンスが届いた瞬間に一気にクリア表示へ切り替える。
- 複数枚生成して選べるようにする（最低3枚程度のグリッド表示）。

## 6. バックエンドAPI仕様

| エンドポイント | メソッド | 概要 |
|---|---|---|
| `/drafts` | POST | 下書き新規作成（`raw_text` を受け取る） |
| `/drafts` | GET | 下書き一覧取得 |
| `/drafts/{draft_id}` | GET | 下書き詳細取得 |
| `/drafts/{draft_id}/extract` | POST | 要点抽出を実行し `extracted_points` を更新 |
| `/generate/blog` | POST | `draft_id` を受け取り、ブログ本文を生成して `generated_texts` に保存 |
| `/generate/crowdfunding` | POST | 同上、クラファン本文 |
| `/generate/image` | POST | `prompt`・`image_type` を受け取りDALL-E APIを呼び出し、結果を `generated_images` に保存 |
| `/generate/image/prompt-draft` | POST | 4.4節のたたき台プロンプトを生成して返す（実際の画像生成はしない） |

すべてのエンドポイントは `tenant_id` をクエリパラメータまたはボディで受け取れるようにするが、MVPでは未指定時は `aiaruku` をデフォルト値として扱ってよい。

GitHub Actions / Cloud Runで以下を設定する想定。

- `ANTHROPIC_API_KEY`

- `GCP_PROJECT_ID`

## 9. 将来構想（今回のスコープ外、設計の余地だけ残す）

`generated_texts.output_type` を文字列型にしておくことで、将来 `youtube_script` / `manga` / `novel` / `video` を追加する際に、データ構造を変更せずプロンプトテンプレートの追加のみで対応できるようにしておく。今回はこれらのプラグインは実装しない。
